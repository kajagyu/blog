html&css 정리

checkbox 사용하기

checkbox는 아래와 같이 사용함
fieldset 태그나 legend 태그는 사용하지 않아도 무방함.
실제 checkbox 태그는 div 안의 태그인데 태그에 대한 속성은 아래와 같음
type: checkbox 타입 지정
id: label의 for 속성과 매치하기 위해 사용
name: 해당 checkbox를 나타내는 속성
value: checkbox 내의 해당 value를 구분하기 위한 속성(서버에서 해당 속성으로 구분)

text 세로 정렬

div로 감싸는 text 세로 정렬 시 해당 div의 height와 line-height를 일치 시킨다
예를 들어 height가 40px이면 line-height도 40px로 맞춤

그러나 위와 같은 방법 적용 시 해당 div를 리스트 형식으로 동시에 호출하는 등엔 문제가 발생함.

그렇기에 그러한 경우엔 아래와 같은 방법을 적용

위의 방법에서 중요한 2가지는 아래와 같음

  1. 텍스트를 감싸는 블록의 height 값이 명확해야 한다.
  2. 부모 앨리먼트(div)의 css 속성을 display:table; 자식 앨리먼트(p)의 css 속성을 display:table-cell; 로 지정해주면 vertical-align 속성이 적용된다.
    (IE7 이하 버전에서는 display:table, table-cell이 지원되지 않기 때문에 적용되지 않는다.)

span을 div처럼 활용하기

span태그는 inline 속성이므로 width, height 지정 불가함.
그러나 ‘display: inline-block;’를 지정하면 span 태그를 div 태그와 같이
width, height 지정 가능함.

사용하고자하는 div (id=intro)가 가로 세로 사이즈가 각각 300, 150 이라고 가정하면..
아래와 같이 div 레이어를 좌측 및 상단으로 절반을 밀어낸후, 화면의 50%영역에 배치하면 중앙에 배치된 효과를 얻을 수 있음.

react 에러 정리

expected this to be used by class method…

에러 문구:
expected this to be used by class method

클래스 메소드에서 this를 사용하지 않을때 발생
-> this 키워드를 사용하여 해결(생성자에서 this.state data를 초기화 하는 등…)

Objects are not valid as a React child..

에러문구:
Objects are not valid as a React child (found: object with keys {}). If you meant to render a collection of children, use an array instead.

rendering 시에 array 형 데이터를 올바르게 파싱하지 않을때 발생
-> render 메소드를 확인하여 array형 데이터를 map을 이용하는 등으로 파싱을 올바르게 하여 출력할것

javascript 정리

URL 경로 파싱하기

URL 경로를 가져와 파싱하고자 한다면 아래와 같이 window 객체를 이용
아래의 예에 대한 설명을 다음과 같음

window.location.pathname
-> URL 경로 가져옴
window.location.pathname.split(‘/‘)
-> URL 경로를 / 로 split하여 배열 데이터로 초기화
pathName[pathName.length - 1]
-> 배열 데이터에 들어있는 URL 경로에서 맨 마지막 경로명을 가져와서 초기화
